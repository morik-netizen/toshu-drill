# UXデザイン仕様

## デザイン方針

ミニマルとゲーミフィケーションの**中間**。シンプルだがランキング・ストリーク・ポイントあり。

## 画面構成

### ナビゲーション（ボトムタブ 4つ）

```
[ホーム] [学習] [間違いノート] [みんなの進捗]
```

※「ランキング」ではなく「みんなの進捗」（心理的圧迫感を軽減）

## モチベーション設計

| 要素 | 仕様 |
|------|------|
| ポイント | 正解+10pt、デイリーチャレンジ全問正解+50pt |
| ストリーク | **週5日以上**でカウント（毎日でなくてOK） |
| ランキング | 前後3人表示 + 3つの軸（総合/伸び率/継続日数） |
| カバー率 | 常にホームに大きく表示 |
| テストカウントダウン | 次のQ1-Q4テストまでの日数 |
| 自己比較 | 先週 vs 今週の正答率グラフ |

### やらないこと

- バッジ・レベルシステム（ごちゃつくため省略）
- ミッション/デイリータスク（「おすすめ○問」で十分）
- 全50人の順位リスト公開
- 順位が下がった通知

## 通知設計

| 通知タイプ | 頻度 | タイミング | 条件 |
|-----------|------|-----------|------|
| デイリーリマインド | 1日1回 | ユーザーが前日開いた時刻の±30分 | 未学習の場合のみ |
| ストリーク危機 | 1日1回 | 21:00 | 今週4日未学習の場合 |
| 順位UP | 不定期 | 即時 | 抜いた時のみ（ポジティブ） |
| テストカウントダウン | テスト3日前+前日 | 18:00 | 常時 |
| ウィークリーレポート | 週1回 | 日曜 18:00 | 常時 |

### 文面の方針

- 良い例: 「今日のチャレンジ5問、準備できてるよ。サクッと解こう」
- 悪い例: 「今日はまだ学習していません。ストリークが途切れます!」
- 初回起動時に通知頻度をユーザーが選択（毎日/週3回/オフ）

## オンボーディング（初回起動）

```
画面1: ウェルカム
  → 名前入力（ニックネーム可）

画面2: 目標設定
  → 「1日何問?」 [5問 / 10問 / 20問]

画面3: 通知設定
  → [毎日 / 週3回 / なし]

画面4: チュートリアル問題
  → 実際に1問解かせる（+10pt獲得）

画面5: ホーム画面追加の案内（PWA）
  → iOS/Android別にスクリーンショット付き手順

→ ホーム画面へ（10pt獲得済みの状態）
```

- 最大5画面、全画面に「スキップ」配置
- 初回で10pt獲得させ「ゼロスタート」を避ける
