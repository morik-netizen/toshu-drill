# 学習設計

## 5つの学習エージェント

### Agent 1: 診断テスト（知らないを発見する）

- 授業進度に連動して、学習済み単元からカテゴリ別正答率マップを生成
- 初回は全カテゴリ一斉ではなく、授業で扱った単元のみから出題

### Agent 2: 克服ドリル（知らない → 知っている へ）

- 弱点カテゴリを集中攻略（1セッション10問）
- 間違えた問題を**反転モード**で再出題（Phase 3）:
  - 方法C: 「正しいもの→誤りを選べ」に変換
  - 方法D: AIが同概念の新問題を生成（教員監修必須）

### Agent 3: 定着復習（忘れる → 覚え続ける）

- **SM-2アルゴリズム**による間隔反復
- 定着基準: **残日数に応じた動的閾値**
  - 早期解放カテゴリ: 5回連続正解
  - 後半解放カテゴリ: 3回連続正解
- 期末3週間は最大間隔を7日に制限
- 定着済み問題も5-10%の確率でランダム出題（偽の定着を防止）

### Agent 4: 模擬試験（混同する → 区別できる）

- Q1-Q4の4回の練習テスト（各40問、合格ライン80%）
- 何度でも受験可（毎回ランダム再抽出）
- 混同しやすい概念を意図的に隣接配置（交互練習）
- 終了後に混同分析レポート

### Agent 5: AIコーチ（Phase 3）

- AWS Bedrock Haiku 4.0（デフォルト）+ Sonnet 4.6（詳細解説）
- 個別の覚え方アドバイス、語呂合わせ、比較解説
- レート制限: 10回/日/人
- AI生成内容には「正確な内容は解説を確認してください」の注意書き

## 出題ロジック

### カバー率優先方式

全393問に1回は触れることを最優先。

```
推奨量 = 未挑戦問題数 ÷ 残日数 + SM-2復習問題（3-5問）
上限: 30問/日
下限: 5問/日
```

### 授業進度連動

- 毎週木曜 朝6:00（授業当日）に新単元を自動解放（午前・午後コース共通）
- 新単元 50-60% + 前回 20-30% + それ以前 10-20%

### 類似問題制御

- 同一概念の類似問題をグループ化
- 同セッション内では1グループから最大1問のみ出題
- パターンマッチング学習への退化を防止

## SM-2アルゴリズム詳細

### 品質評価（簡易版 - Phase 1）

| 結果 | quality |
|------|---------|
| 正解 | 4 |
| 不正解 | 1 |

### 品質評価（詳細版 - Phase 2以降）

| 結果 | quality |
|------|---------|
| 即答正解（<10秒） | 5 |
| 正解（10-30秒） | 4 |
| 迷って正解（>30秒） | 3 |
| 不正解（正答を覚えていた） | 2 |
| 不正解（正答も覚えていない） | 1 |

### 復習間隔

```
1回目正解 → 1日後
2回目正解 → 3日後
3回目正解 → 7日後
4回目正解 → 14日後
5回目正解 → 定着済み

間違えた場合 → 間隔リセット → 翌日
```

## 問題の状態遷移

```
🔒 ロック中   → 授業で扱っていない単元
🆕 未挑戦     → 解放済み、まだ未回答
❌ 学習中     → 間違えたことがある
🔄 復習中     → SM-2で間隔反復中
✅ 定着済み   → 動的閾値を達成（3-5回連続正解）
```

## デイリーチャレンジ

- 毎日5問を全員共通で出題
- 全問正解でボーナスポイント
- 「今日の正答率 62%」のような共有体験を生成
