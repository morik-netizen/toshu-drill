# リスクと対策

## マルチエージェントレビューで特定されたリスク

### 開発リスク

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| 4/16に間に合わない | 高 | 構成をシンプル化（DynamoDB→PostgreSQL, Cognito→NextAuth.js）。AIコーチはPhase 3に移行 |
| Cognito OAuth設定に時間がかかる | 高→解消 | NextAuth.jsに変更。設定が圧倒的に簡単 |
| DynamoDBの集計クエリが困難 | 高→解消 | PostgreSQLに変更。SQLで自然に集計可能 |
| 教員1人での開発・運用 | 中 | IaC（CDK/SAM）でワンコマンド再デプロイ。操作マニュアル作成 |

### 教育リスク

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| パターンマッチング学習への退化 | 高 | 類似問題グループ化、同セッション内1グループ1問制限 |
| SM-2で後半問題が定着に到達不能 | 高 | 動的閾値（後半3回、期末は最大間隔7日制限） |
| 推奨量過多による認知負荷 | 中 | 1日上限30問。推奨量を自動計算 |
| 授業欠席者への対応不足 | 中 | カテゴリ解放時に概要説明を提供。AIコーチで補足（Phase 3） |
| 期末の詰め込み学習 | 中 | Q1時点でカバー率低い学生に教員アラート。ペース予測表示 |
| ストリーク途切れの「もういいや」効果 | 中 | 週5日以上の柔軟な定義。復帰ボーナス |
| AI語呂合わせの品質リスク | 低 | 注意書き付与。頻出論点は教員作成の公式語呂合わせを優先 |

### UXリスク

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| ランキング下位の学生のモチベーション低下 | 高 | 前後3人のみ表示。複数軸（伸び率、継続日数） |
| 2日目に戻ってこない（離脱D） | 高 | 初日に10pt獲得。翌日24時間後に通知。デイリーチャレンジ |
| PWAインストールのハードル | 中 | オンボーディングでスクリーンショット付き案内 |
| 通知疲れ | 中 | 初回に頻度を選択（毎日/週3/オフ）。1日最大1通 |
| 学生が使わない | 中 | 終講試験の成績評価に「小テスト40%」として組み込み（既にシラバスに記載） |

### 技術・運用リスク

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| AWSコスト超過 | 低 | AWS Budgets設定（$200/月警告、$800累計緊急） |
| Bedrock API一時停止 | 低 | AIコーチ以外は動作継続。CSVの静的解説にフォールバック |
| 授業日にアプリが落ちる | 中 | レベル別緊急計画。PDF問題集をバックアップ準備 |
| 学生データ漏洩 | 中 | HTTPS、サーバーサイドDB接続、Prismaパラメータ化クエリ |
| 問題データの著作権 | 低 | 国家試験過去問は公開情報。解説は教員のオリジナル |
| 学校IT管理者の承認 | 中 | Google Workspace OAuth連携の事前承認を早めに取得 |

### 緊急対応計画

| レベル | 状況 | 対応 |
|--------|------|------|
| レベル1 | AIコーチのみ停止 | CSVの静的解説を表示。問題演習は継続 |
| レベル2 | バックエンド全体停止 | 事前エクスポートしたPDF問題集を配布 |
| レベル3 | フロントエンドも停止 | 紙ベースの問題で授業継続 |
